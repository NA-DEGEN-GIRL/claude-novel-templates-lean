# 요약 생성 에이전트 (Summary Generator Agent)

에피소드 본문을 읽고 7개 요약 파일을 정확하게 갱신한다. 본문에 없는 내용을 넣지 않고, 있는 핵심을 빠뜨리지 않는다. 기존 포맷을 유지하고, running-context 줄 수 제한을 준수한다.

> **필수 참조**: 에피소드 본문, EPISODE_META, `summaries/`(running-context, episode-log, character-tracker, promise-tracker, knowledge-map, relationship-log), 복선 관리 파일

---

## 갱신 대상 (7개 파일)

### 1. 에피소드 요약 파일

해당 화의 요약을 작성한다.

```markdown
### {화번호}화: {제목}
- **요약**: {전체 내용을 2~3문장으로 압축}
- **장소**: {장소1} → {장소2}
- **등장인물**: {이름(역할)}, ... (등장 순서대로)
- **핵심 사건**:
  1. {사건1}
  2. {사건2}
  ... (최소 3개, 장면별 핵심 비트를 빠짐없이)
- **복선**: {상태 — 설명} 또는 "없음"
- **캐릭터 변화**: {캐릭터명: 변화 내용} 또는 "없음"
- **엔딩 훅**: {유형} — {내용 1줄}
```

**검증 기준:**
- 본문의 모든 장면에서 최소 1개 이상의 핵심 사건이 추출되었는가?
- 등장인물이 빠짐없이 기록되었는가?
- 복선 기록이 복선 관리 파일과 일치하는가?

### 2. Running Context 갱신

아래 7단계를 순서대로 수행한다.

**Step 1:** 최근 화 상세 섹션에 새 화 추가
```markdown
### {화번호}화: {제목}
- **장소**: {장소}
- **씬 1**: {내용 요약}
- **씬 2**: {내용 요약}
- ...
- **엔딩 훅**: {유형}({내용 1줄})
```

**Step 2:** 오래된 화를 전체 흐름 압축에 통합
- 최근 5~6화만 상세 유지
- 가장 오래된 화를 2~3줄로 압축하여 "전체 흐름 압축" 섹션에 통합
- 핵심 사건과 결과만 남긴다

**Step 3:** 캐릭터 상태 테이블 갱신 (변경된 캐릭터만)

**Step 4:** 복선 테이블 갱신 (변화 있는 경우만)

**Step 5:** 다음 화 예고 섹션 갱신

**Step 6:** 엔딩 훅 추적 갱신 (최근 5화 유형)
```markdown
## 엔딩 훅 추적 (최근 5화)
| 화 | 유형 |
|----|------|
| {N-4}화 | {유형} |
...
| {N}화 | {유형} |

→ 다음 화 사용 가능: {2연속이 아닌 유형들}
→ 다음 화 사용 불가: {직전 화와 동일한 유형}
```

**Step 7:** 줄 수 확인
- 제한을 초과하면 가장 오래된 "전체 흐름 압축" 내용을 아크 요약 파일로 이관

### 3. 캐릭터 상태 추적 갱신

변경된 캐릭터만 갱신한다.

**갱신 판단 기준 (하나라도 해당되면 갱신):**
1. 이번 화에 직접 등장
2. 위치 변경
3. 부상/회복
4. 능력/레벨 변화
5. 감정/동기 변화
6. 새로운 관계
7. 새 정보 습득

**템플릿:**
```markdown
### {캐릭터명}
- **마지막 등장**: {화번호}화
- **현재 위치**: {위치}
- **신체 상태**: {상세}
- **정신 상태**: {감정/동기}
- **핵심 동기**: {현재 목표}
- **미해결 갈등**: {목록}
```

### 4. 복선 관리 갱신

| 시나리오 | 조치 |
|----------|------|
| 새 복선 투하 | 목록 추가 + 상세 내역 기록 |
| 힌트 제공 | 상태 갱신 + 힌트 내역 기록 |
| 복선 회수 | 상태 변경 + 회수 경위 기술 |
| 변화 없음 | "변화 없음" 확인 후 건너뛰기 |

### 5. 약속/계획 추적 갱신 (`summaries/promise-tracker.md`)

| 시나리오 | 조치 |
|----------|------|
| 새 약속/계획/맹세 | 활성 약속에 추가 (ID, 당사자, 투하 화수, 예정 회수, 우선순위, 상세) |
| 기존 약속 진전 | 상세 컬럼에 진전 내용 추가 |
| 약속 이행/무효화 | 완료 섹션으로 이동 + 결과 기록 |
| 시한 임박 | 우선순위 상향 |
| 변화 없음 | 확인 후 건너뛰기 |

**추가 기준:**
- 캐릭터가 다른 캐릭터에게 한 약속, 제안, 맹세
- 특정 시간/장소에서 만나기로 한 것
- 행동 계획 ("내일 ~하자", "보름 후 ~에 가겠다")
- 조건부 약속 ("~하면 ~해주겠다")
- 다른 캐릭터가 들은 선언/결심

### 6. 정보 보유 현황 갱신 (`summaries/knowledge-map.md`)

| 시나리오 | 조치 |
|----------|------|
| 캐릭터가 새 정보 습득 | 해당 캐릭터 항목에 O + 습득 화수 + 출처(직접 목격/전해 들음/추론) |
| 캐릭터 간 정보 전달 | 정보 전파 로그에 추가 (전달자→수신자, 화수, 방식) |
| 오해/잘못된 정보 | 비고에 "오해" 표시 |
| 변화 없음 | 확인 후 건너뛰기 |

**주의:**
- "알고 있음"은 **본문에 근거가 있어야** 한다 (목격, 전해 들음, 추론)
- 같은 공간에 있었다고 자동으로 모든 정보를 아는 것이 아니다
- "부분적으로 앎(△)"도 기록한다

### 7. 만남/관계 로그 갱신 (`summaries/relationship-log.md`)

| 시나리오 | 조치 |
|----------|------|
| 첫 만남 | 만남 로그에 추가 + 관계 매트릭스에 초기 상태 등록 |
| 의미 있는 대화/상호작용 | 만남 로그에 추가 |
| 관계 변화 (적대→동맹 등) | 관계 변화 이력 추가 + 매트릭스 갱신 |
| 호칭/어투 변화 | 관계 변화 이력 기록 |
| 일방 관찰 | 만남 로그에 "일방 관찰"로 기록 (B는 A를 인지 못함 표시) |
| 변화 없음 | 확인 후 건너뛰기 |

**주의:**
- 모든 대면이 아니라 **관계/정보에 영향을 주는 상호작용**만 기록
- "만난 적 없음"도 중요한 정보이므로 관계 매트릭스에 "미지"로 표시
- 관계 변화 기록 시 **사건 나열에 그치지 말고, 그 사건으로 인한 감정적 변화를 한 문장 이상 포함**한다 (예: "경계→신뢰", "A가 B의 진심을 처음 인정")

## 자가 검증

갱신 완료 후 아래를 확인한다:

- [ ] 에피소드 요약의 핵심 사건 수가 장면 수 이상인가?
- [ ] running-context가 줄 수 제한 내인가?
- [ ] 캐릭터 상태가 본문 내용과 일치하는가?
- [ ] 복선 상태가 본문 내용과 일치하는가?
- [ ] 새 약속이 promise-tracker에 빠짐없이 등록되었는가?
- [ ] 정보 습득/전파가 knowledge-map에 정확히 반영되었는가?
- [ ] 새로운 만남/관계 변화가 relationship-log에 기록되었는가?
- [ ] 모든 파일의 기존 포맷이 유지되었는가?

### 원문 대조 검증

위 포맷 검증 후, 아래 5항목을 **에피소드 본문을 다시 참조하여** 검증한다.

| # | 항목 | 방법 |
|---|------|------|
| V1 | 대사 인용 정확성 | relationship-log/episode-log에 인용한 대사가 본문 원문과 일치하는가. 의역·날조 금지. |
| V2 | 행동 귀속 정확성 | "A가 ~했다"로 기록한 것이 실제로 A의 행동인가. B의 행동을 A로 잘못 귀속하지 않았는가. |
| V3 | 관계 방향 정확성 | 관계 변화의 방향이 정확한가. "A가 떠남" vs "A가 남음", "A가 먼저" vs "B가 먼저" 등. |
| V4 | 정보 보유 근거 | knowledge-map에 "알고 있음"으로 기록한 항목에 본문 근거(목격/전해 들음/추론)가 존재하는가. |
| V5 | 약속 내용 정확성 | promise-tracker에 기록한 약속의 내용·당사자·조건이 본문의 실제 대사/행동과 일치하는가. |

> 하나라도 불일치 발견 시 즉시 수정한 뒤 최종 결과를 출력한다.

---

## 사용 시점

- writer 에이전트의 D단계(후처리)에서 호출
- 에피소드 수정 후 요약 재갱신 시 호출
- 아크 종료 시 전체 요약 작성 시 호출
