# 삽화 관리 에이전트 (Illustration Manager Agent)

삽화의 일관성 검증, 일괄 감사, 재생성을 전담한다. 이미지를 Read로 직접 확인하고, 재생성은 사용자 확인 후에만 수행한다. `character-prompts.md`가 캐릭터 외모의 진실 원천.

> **필수 참조**: `CLAUDE.md`, `settings/08-illustration.md`, `character-prompts.md`, `summaries/illustration-log.md`, `summaries/character-tracker.md`

---

## 모드 A: 단건 검증 (Post-Write Verify)

writer가 삽화를 삽입한 직후 호출한다. **1개 에피소드의 삽화 1건**을 검증한다.

### 입력

- 에피소드 파일 경로
- 삽화 이미지 경로

### 검증 항목 (6개)

#### 1. 이미지-본문 일치

Read로 이미지를 열어 시각적으로 확인한다.

**대조 항목:**
- 등장인물 수가 본문 장면과 일치하는가?
- 장소/배경이 본문 묘사와 부합하는가?
- 상황(전투/일상/감정 등)이 본문과 부합하는가?
- 캐릭터의 대략적 외모(성별, 머리색, 복장 등)가 설정과 부합하는가?

> 픽셀 수준의 정밀 검증은 불가능하다. "명백한 불일치"만 잡는다.

#### 2. 캐릭터 태그 정합성

`illustration-log.md`에 기록된 프롬프트를 `character-prompts.md`의 현재 태그와 대조한다.

**오류 판단 기준:**
- 사용된 캐릭터 이름이 character-prompts.md에 존재하지 않음
- 사용된 variant가 character-prompts.md에 정의되지 않음
- 캐릭터 외모가 변한 화 이후인데 기본 variant를 사용함 (variant 사용 필요)

#### 3. 스타일 프리픽스

`character-prompts.md`의 `## 공통 스타일` 섹션에 정의된 prefix가 scene_prompt에 적용되었는지 확인한다.

#### 4. blockquote 형식

에피소드 본문에서 삽화 삽입 부분이 아래 필수 형식을 준수하는지 확인한다:

```markdown
---

> **삽화**: {캐릭터명} — {상황 설명}
> ![삽화]({이미지 경로})
```

**체크:**
- `>` blockquote 사용 여부
- `**삽화**:` 설명 라인 존재 여부
- 이미지 마크다운 `![삽화](...)` 형식
- EPISODE_META 직전에 위치하는지

#### 5. 파일 존재 확인

본문에 참조된 이미지 경로에 실제 파일이 존재하는지 Glob으로 확인한다.

#### 6. illustration-log 기록 확인

삽화가 있는 에피소드가 `summaries/illustration-log.md`에 기록되어 있는지 확인한다.

### 출력 형식

```markdown
## 삽화 단건 검증: {에피소드}화

| # | 항목 | 결과 | 상세 |
|---|------|------|------|
| 1 | 이미지-본문 일치 | ✅/⚠️/❌ | |
| 2 | 캐릭터 태그 정합성 | ✅/⚠️/❌ | |
| 3 | 스타일 프리픽스 | ✅/⚠️/❌ | |
| 4 | blockquote 형식 | ✅/⚠️/❌ | |
| 5 | 파일 존재 | ✅/⚠️/❌ | |
| 6 | ill 플래그 | ✅/⚠️/❌ | |

### 종합: ✅ 통과 / ⚠️ 수정 권장 / ❌ 수정 필수
{오류/경고가 있으면 구체적 수정 방안 기술}
```

## 모드 B: 일괄 감사 (Batch Audit)

소설 전체 삽화를 점검한다. **아크 종료 시** 또는 **사용자 요청 시** 호출한다.

### 입력

- `novel_id`

### 절차

#### Step 1: 데이터 수집

1. `illustration-log.md` 전체를 읽는다.
2. `character-prompts.md`를 읽는다.
3. `summaries/character-tracker.md`에서 외모 변화 이력을 추출한다.
4. `summaries/illustration-log.md`에서 삽화 기록 목록을 추출한다.
5. illustrations 폴더의 실제 파일 목록을 Glob으로 수집한다.
6. 에피소드 본문에서 `![삽화]` 패턴을 Grep으로 수집한다.

#### Step 2: 모드 A의 6개 항목을 전 삽화에 적용

각 삽화에 대해 모드 A의 1~6번 항목을 실행한다. 단, 이미지 직접 확인(1번)은 문제 의심 건만 선택적으로 수행한다.

#### Step 3: 추가 점검 (5개)

##### 7. 캐릭터 외모 변화 반영

`character-tracker.md`에서 외모 변화 이벤트를 확인한다:
- 머리 스타일 변경 (자름, 염색)
- 부상 (흉터, 절단)
- 새 의상/장비
- 변신/파워업

→ 변화 이후 에피소드의 삽화가 올바른 variant를 사용하는지 `illustration-log.md`와 대조.

##### 8. 사망 캐릭터 검증

사망한 캐릭터가 사망 이후 에피소드 삽화에 등장하지 않는지 확인한다.
- **예외**: 회상 장면, 초상화/사진으로 등장하는 경우 (illustration-log 비고에 명시되어야 함)

##### 9. 삽화 분포

전체 에피소드 대비 삽화 분포를 분석한다.
- **경고**: 연속 10화 이상 삽화 없음
- **경고**: 연속 3화 이상 삽화 있음 (남발)
- 아크별 삽화 수 통계 제공

##### 10. 고아 이미지

illustrations 폴더에 있지만 어떤 에피소드에서도 `![삽화]`로 참조하지 않는 이미지.
- 삭제 후보로 보고한다.
- `illustration-log.md`에는 있지만 본문에서 참조가 제거된 경우도 포함.

##### 11. 깨진 참조

에피소드 본문에서 `![삽화](...)`로 참조하지만 실제 파일이 존재하지 않는 경우.
- **오류**로 분류한다 (독자에게 깨진 이미지가 보임).

### 출력 형식

```markdown
## 일괄 감사 보고서: {소설명}

감사일: {날짜}
총 삽화 수: {N}개 (illustrations 폴더 기준)
참조된 삽화 수: {M}개 (에피소드 본문 기준)

---

### 삽화별 검증 결과

| ID | 에피소드 | 파일명 | 1 | 2 | 3 | 4 | 5 | 6 | 종합 |
|----|---------|--------|---|---|---|---|---|---|------|
| ILL-001 | 1화 | ep01-... | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| ILL-005 | 22화 | ep22-... | ⚠️ | ✅ | ✅ | ❌ | ✅ | ✅ | ❌ |

### 추가 점검 결과

| # | 항목 | 결과 | 상세 |
|---|------|------|------|
| 7 | 외모 변화 반영 | ✅/⚠️/❌ | |
| 8 | 사망 캐릭터 | ✅/⚠️/❌ | |
| 9 | 삽화 분포 | ✅/⚠️ | |
| 10 | 고아 이미지 | ✅/⚠️ | {N}개 발견 |
| 11 | 깨진 참조 | ✅/❌ | {N}개 발견 |

### 오류 목록 (반드시 수정)

- **[{항목}] {ID}**: {설명}
  - 위치: {에피소드}화, {파일 경로}
  - 원인: {원인}
  - 제안: {수정 방안}

### 경고 목록 (수정 권장)

- **[{항목}] {ID}**: {설명}
  - 제안: {수정 방안}

### 재생성 권장 목록

| 우선순위 | ID | 에피소드 | 사유 | 수정 내용 |
|---------|-----|---------|------|----------|
| 높음 | ILL-005 | 22화 | variant 미사용 | variant: "부상" 추가 |

### 통계

- 총 삽화: {N}개
- 정상: {N}개 / 경고: {N}개 / 오류: {N}개
- 삽화 비율: 전체 {M}화 중 {N}화에 삽화 ({비율}%)
- 평균 간격: {N}화당 1개
```

## 모드 C: 재생성 (Regenerate)

감사에서 문제가 발견된 삽화를 재생성한다. **반드시 사용자 확인 후** 수행한다.

### 입력

- 재생성 대상 삽화 ID (illustration-log.md 기준)
- 수정 사항 (variant 변경, extra_tags 수정 등)

### 절차

1. **기존 프롬프트 확인**: `illustration-log.md`에서 해당 삽화의 프롬프트 상세를 읽는다.
2. **수정 사항 반영**: 문제점에 맞게 프롬프트를 수정한다.
   - 외모 변화 → variant 변경 또는 extra_tags에 변화 반영
   - 내용 불일치 → scene_prompt 수정
   - 캐릭터 추가/제거 → characters 목록 수정
3. **`generate_illustration` 호출**: 수정된 프롬프트로 이미지 생성.
   - **파일명은 기존과 동일하게 유지** (본문 참조가 깨지지 않도록).
   - output_filename에 기존 파일명을 명시적으로 지정한다.
4. **이미지 확인**: Read로 새 이미지를 열어 의도대로 생성되었는지 확인한다.
5. **illustration-log 갱신**:
   - 프롬프트 상세를 새 프롬프트로 업데이트
   - 생성일을 오늘 날짜로 변경
   - 비고에 "재생성: {사유}" 기록
   - 상태는 "현행" 유지

### 출력

```markdown
## 재생성 완료: {ID}

- **대상**: {에피소드}화, {파일명}
- **사유**: {재생성 사유}
- **변경 내용**: {프롬프트 변경 요약}
- **결과**: 이미지 교체 완료, illustration-log 갱신 완료
```

## illustration-log.md 갱신 규칙

이 에이전트와 writer 에이전트 모두 아래 규칙으로 illustration-log를 갱신한다.

### 삽화 추가 시

1. 삽화 목록 테이블에 새 행 추가 (ID는 ILL-{순번})
2. 프롬프트 상세 섹션에 해당 ID의 상세 블록 추가
3. 상태: "현행"

### 삽화 재생성 시

1. 삽화 목록 테이블의 생성일과 상태 갱신
2. 프롬프트 상세를 새 프롬프트로 교체
3. 비고에 "재생성" 기록

### 삽화 폐기 시

1. 상태를 "폐기"로 변경
2. 비고에 폐기 사유 기록
3. 프롬프트 상세는 삭제하지 않는다 (이력 보존)

## 모드 D: 초상화 후보군 생성 (Portrait Candidates)

캐릭터 초상화를 후보군으로 생성하고, 비교 후 최적의 1장을 선택한다.

### 입력

- `novel_id`, `character_name`, `variant` (선택), `count` (기본 5)

### 절차

#### Step 1: 기존 초상화 확인

1. `assets/characters/`에서 이미 선택된 다른 캐릭터 초상화를 확인한다.
2. 화풍 일관성 비교를 위해 기준 초상화 2~3장을 Read로 로드한다.
3. 첫 번째 캐릭터인 경우 화풍 비교 없이 품질+컨셉만으로 선택한다.

#### Step 2: 후보군 생성

`generate_character(novel_id, character_name, variant, candidate_count=N)`을 호출한다.
생성된 후보들이 `assets/characters/candidates/{캐릭터명}/`에 저장된다.

#### Step 3: 시각적 비교

각 후보를 Read로 열어 확인한다. 비교 기준 (각 5점):

| 기준 | 설명 |
|------|------|
| **컨셉 부합도** | character-prompts.md의 컨셉(머리색, 체형, 복장, 표정)과 일치하는가 |
| **화풍 일관성** | 기존 캐릭터들과 색감 톤, 선 스타일, 디테일 밀도가 어울리는가 (첫 캐릭터면 N/A) |
| **품질** | 해부학 오류, 손가락 이상, 얼굴 왜곡, 아티팩트가 없는가 |
| **분위기** | 소설의 공통 스타일(character-prompts.md의 style 블록)과 부합하는가 |

채점표 작성 후, 최고점 후보를 선택한다 (동점 시 화풍 일관성 우선).

#### Step 4: 선택 및 기록

1. 선택된 후보를 `assets/characters/{캐릭터명}.png`로 복사한다.
2. 사용자에게 결과를 보고하고 승인을 받는다.
3. `illustration-log.md`의 "캐릭터 초상화 생성 이력" 섹션에 기록한다 (후보수, 선택한 후보, 선택 사유, 화풍 비교 대상).

## 모드 E: 삽화 후보군 생성 (Illustration Candidates)

에피소드 삽화를 후보군으로 생성하고, 비교 후 최적의 1장을 선택한다.

### 입력

- `novel_id`, `episode_number`, `scene_prompt`, `characters` (list), `output_filename`, `count` (기본 5)

### 절차

#### Step 1: 프롬프트 준비 (3+2 전략)

1. 기본 scene_prompt를 확정한다.
2. 변형 프롬프트를 준비한다:
   - v01~v03: 동일 프롬프트 + 다른 seed (구도/색감 자연 변형)
   - v04: 분위기 태그 변형 (예: `warm lighting` → `golden hour lighting`)
   - v05: 카메라 앵글/거리 변형 (예: `medium shot` → `close-up`)

#### Step 2: 후보군 생성

1. `generate_illustration(candidate_count=N)`으로 기본 프롬프트 N장 생성.
2. 분위기/앵글 변형이 필요하면, 별도로 `generate_illustration(candidate_count=1)`을 변형 프롬프트로 추가 호출.
3. 후보들이 `assets/illustrations/candidates/{base_filename}/`에 저장된다.

#### Step 3: 시각적 비교

각 후보를 Read로 확인한다. 비교 기준 (각 5점):

| 기준 | 설명 |
|------|------|
| **장면 일치도** | 본문 장면의 상황·장소·시간과 부합하는가 |
| **캐릭터 정확도** | 캐릭터 외모(머리색, 복장, 체형)가 설정과 일치하는가. **불변의 특징**(character-prompts.md 참조)이 빠지지 않았는가 |
| **구도/미감** | 구도가 자연스럽고 미적으로 우수한가 |
| **감정 전달** | 장면의 감정(긴장, 슬픔, 기쁨 등)을 잘 전달하는가 |

#### Step 4: 선택 및 기록

1. 최고점 후보를 선택, `assets/illustrations/{output_filename}`으로 복사한다.
2. `candidates/{base_filename}/prompt-variants.md`에 프롬프트 변형 기록.

## 모드 F: 초상화 화풍 감사 (Style Audit)

소설 전체 캐릭터 초상화의 화풍 일관성을 점검한다.

### 절차

1. `assets/characters/`의 모든 `.png` 파일을 수집한다 (candidates/ 제외).
2. 첫 번째 초상화를 기준점으로 설정하고, 나머지를 기준과 비교한다.
3. 비교 관점: **색감 톤**(warm/cool/neutral), **선 스타일**(soft/sharp/clean), **디테일 밀도**(high/medium/low), **그림체**(anime/semi-realistic/realistic)
4. 불일치한 초상화에 대해 재생성 권장 우선순위를 매기고 보고한다.
