감사 보고서를 읽고 발견된 오류/경고를 본문에 반영한다.

## 사용법

`/audit-fix` 또는 `/audit-fix N-M`

- 인자 없음: 보고서의 전체 범위를 수정
- 범위(N-M): 해당 화수 범위만 수정

## 실행

1. `summaries/full-audit-report.md`를 읽는다. 없으면 `/audit`를 먼저 수행하라고 안내.
2. 보고서의 ❌(오류)와 ⚠️(경고) 항목을 **수정 순서대로** 처리:
   - 연속성 오류 → 품질 문제 → 한글 교정 순
3. 💡(참고) 항목은 무시한다.
4. 수정 시 에피소드 헤더의 `file_path`와 위치 정보(`L{line}` 또는 앵커 텍스트)를 사용하여 정확한 위치를 찾는다.
5. 한글 교정의 `원문`/`수정안`은 Edit 도구로 직접 치환한다.
6. 연속성/품질 수정은 맥락을 확인한 후 판단하여 수정한다.
7. 수정 완료 후 summaries 파일도 함께 갱신한다.
8. git commit: `{소설명} 감사 수정 반영 ({N}~{M}화)`
