감사 보고서(`summaries/full-audit-report.md`)를 읽고 발견된 오류/경고를 본문에 반영한다.

## 사용법

`/audit-fix` 또는 `/audit-fix N-M` 또는 `/audit-fix --resume`

- 인자 없음: 보고서의 전체 범위를 **10화 단위 배치**로 자동 처리
- 범위(N-M): 해당 화수 범위만 수정
- `--resume`: 이전 중단 지점부터 재개 (`full-audit-fix-log.md` 참조)

## 사전 조건

1. `summaries/full-audit-report.md`가 존재해야 한다. 없으면 `/audit`을 먼저 실행하라고 안내.
2. `CLAUDE.md`를 읽어 소설별 규칙(금지사항, 호칭 매트릭스 등)을 확인한다.

## 심각도별 처리

| 심각도 | 처리 |
|--------|------|
| ❌ 오류 | **반드시 수정** |
| ⚠️ 경고 | 원문 확인 후 수정 (맥락상 의도적이면 스킵) |
| 💡 참고 | **수정하지 않음** |

## 배치 처리 (핵심)

> **대량 수정 시 컨텍스트 포화를 방지하기 위해 반드시 배치로 나눈다.**

### 배치 단위

1. 보고서에서 수정 대상 에피소드 목록을 추출한다
2. **10화 단위**로 배치를 나눈다 (예: 1-10화, 11-20화, 21-30화)
3. 아크 경계가 배치 중간에 오면 아크 경계에서 끊는다

### 배치별 처리

각 배치에 대해:

1. **해당 배치의 에피소드만 읽는다** (전체를 한번에 로딩하지 않음)
2. 보고서에서 해당 화수 범위의 항목만 추출
3. Step 1-6 수행 (아래 참조)
4. **배치 완료 후 중간 저장**:
   - 수정된 파일 저장
   - `full-audit-fix-log.md`에 배치 결과 append
   - git commit: `{소설명} 감사 수정 ({N}~{M}화)`
5. **다음 배치로 이동** — 이전 배치의 수정 내용을 새로 읽어 반영

### 한글 교정 앵커 무효화 방지

> 이전 배치에서 수정한 파일의 위치 정보가 이후 배치에서 무효화될 수 있다.

- **같은 에피소드 내**: Step 1(연속성) → Step 2(품질) → Step 3(한글) 순서 엄수. 앞 단계에서 수정한 후 **파일을 다시 읽어** 현재 텍스트 기준으로 한글 교정 앵커를 재탐색
- **다른 에피소드 간**: 배치 내에서 에피소드 번호순 처리하므로 간섭 없음
- **보고서의 `L{line}` 라인 번호가 틀리면**: 앵커 텍스트(`"…5chars**원문**5chars…"`)로 폴백. 앵커도 실패 시 `보류`

## 수정 순서 (에피소드별)

각 에피소드 내에서:

### Step 1. 연속성 오류 수정

보고서 연속성 섹션의 ❌/⚠️를 처리한다.

- 보고서의 `위치`(라인 힌트/앵커)로 해당 구간을 찾는다
- 관련 settings/ 파일과 이전 에피소드를 참조하여 수정
- **스킵 기준** (하나라도 해당 시 → `보류` 기록):
  - 해결안이 2개 이상 가능하여 저자 의도 추정 필요
  - 수정이 대상 범위를 넘어 여러 화 연쇄 수정 필요
  - settings/ 파일 자체를 변경해야 하는 경우
  - 보고서 근거가 약하여 실제 오류인지 불명확
  - 하나의 항목을 수정하면 다른 보고된 항목과 새 충돌이 발생하는 경우 → 두 항목 모두 `보류`로 기록하고 상호 참조

### Step 2. 품질 항목 수정

보고서 품질 섹션의 ❌/⚠️를 처리한다.

- Show don't tell 위반, 중복 묘사 → 해당 구간 rewrite
- AI 심리 패턴 위반 → 맥락 확인 후 수정
- **캐릭터 말투, 의도적 비문, 작가 문체는 존중**
- 판단 어려운 항목 → `보류`

### Step 3. 한글 교정 수정

보고서 한글 교정 섹션의 `원문`/`수정안` 테이블을 적용한다.

- **Step 1-2에서 수정이 발생한 에피소드는 파일을 다시 읽은 후** 한글 교정 적용
- **단일 매칭**: 원문이 파일 내 정확히 1회 → Edit 도구로 즉시 치환
- **다중 매칭**: 보고서의 위치 정보(L{line}, 앵커)로 정확한 위치 특정 후 적용. 특정 불가 → `보류`
- **매칭 실패**: 이미 수정됨 또는 원문 변경 → 스킵, 로그 기록
- 치환 후 해당 **단락 전체**를 읽어 자연스러움 확인

### Step 4. 자연스러움 수정 (Phase 2.5 결과)

보고서의 **자연스러움** 섹션을 처리한다. `/naturalness-fix`와 동일한 판정 기준:

- 서술/나레이션에서 정말 어색하고 의도된 문체가 아닌 것 → **수정**
- 캐릭터 고유 말투, 시적 표현, 의도적 문체, 라이트모티프 → **기각**
- 판단 어려움 → **보류**
- **보고서의 수정안을 그대로 쓰지 않아도 된다.** 문맥에 맞는 더 나은 표현 사용 가능. 단, **최소한의 수정만.** 스타일 폴리싱이 아니라 어색함 제거가 목적. 문장의 리듬과 서브텍스트를 보존할 것.
- 캐릭터 대사 안의 지적은 해당 캐릭터의 말투 설정을 확인한 후 판단.
- Step 3(한글 교정)에서 이미 수정된 부분은 건너뛴다 (중복 수정 방지).

### Step 5. 재검수 (rewrite 발생 시만)

Step 1-2 또는 Step 4에서 **문장이 rewrite된 에피소드**에 한해, `unified-reviewer`를 `continuity` 모드로 호출하여 재검수한다.

- 단순 치환만 된 화(Step 3의 1:1 치환만 적용)는 재검수 불필요
- 재검수에서 새 오류 발견 시 즉시 수정

### Step 6. 요약 갱신

본문 수정이 있었던 에피소드에 대해 **변경된 내용에 관련된 summaries 파일만** 갱신한다:

- 연속성 수정 시: `character-tracker.md`, `knowledge-map.md`, `relationship-log.md`, `promise-tracker.md` 중 해당 파일
- 품질 수정 시: `episode-log.md` (요약 변경 시)
- 한글 교정만: summaries 갱신 불필요

각 배치 완료 시 `summaries/full-audit-fix-log.md`에 append:

```markdown

> 대상: {N}화 ~ {M}화
> 기반 보고서: full-audit-report.md

## 배치 {K}: {N}화 ~ {M}화

| 화수 | 적용 | 스킵 | 보류 |
|------|------|------|------|
| {N}화 | {N}건 | {N}건 | {N}건 |

### 보류 항목

| 화수 | 심각도 | 카테고리 | 항목 | 보류 사유 |
|------|--------|---------|------|----------|
```

## 재개 (`--resume`)

1. `full-audit-fix-log.md`를 읽어 마지막 처리된 배치/화수를 확인
2. 다음 배치부터 재개
3. 이미 처리된 에피소드는 건너뜀

## 커밋

- **배치마다** 커밋 (전체 완료 후가 아님)
- 변경 없으면 커밋하지 않음
- 메시지: `{소설명} 감사 수정 반영 ({N}~{M}화)`

## 수정 원칙

1. **보고서 기반만** — 보고서에 없는 자체 판단 수정 금지. 해당 항목 해결에 필요한 최소 주변 수정만 허용
2. **의도적 표현 존중** — 캐릭터 말투, 사투리, 의도적 비문은 건드리지 않는다
3. **범위를 넘는 판단이 필요하면 보류** — 무리하게 수정하지 않는다
4. **순서 준수** — 연속성 → 품질 → 한글 교정 → 자연스러움 → (rewrite 시) 재검수
5. **배치 단위 처리** — 10화씩 끊어서 처리. 배치마다 커밋.
